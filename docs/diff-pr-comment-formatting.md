# Diff PR Comment Formatting (D3-2)

Updated: 2026-03-29
Owner: Maintainer
Related ticket: `T-20260328-167`

## Purpose

This document defines the markdown formatting layer that renders a
`DiffPRCommentPayload` (G2-1) into a PR comment body string.

It specifies:
- the markdown template structure for PR comment output
- how groups, findings, and severity are rendered
- how trace references are formatted and linked
- the concise vs full rendering modes

It does not specify:
- the `DiffPRCommentPayload` shape (G2-1)
- how the rendered string is posted to a GitHub PR (G2 integration layer)
- terminal / CLI rendering (E2-1)
- regression baseline for semantic summary content (D3-3)

---

## 1. Input and output

**Input**: `DiffPRCommentPayload` (from G2-1 `buildDiffPRCommentPayload()`)
**Output**: `string` — a GitHub-flavored markdown PR comment body

The formatter has **no knowledge** of `DiffCompareResult`, `DiffNarrativeResult`,
or any internal Epic C type.

---

## 2. Comment structure

A rendered PR comment has the following structure:

```
<attention banner> (if containsAmbiguity or headlineSeverity === s3-critical)
<headline>
<group block> × N
<truncation notice> (if truncation.truncated)
<footer>
```

### 2.1 Attention banner (conditional)

```markdown
> ⚠️ **Ambiguity detected** — one or more findings may be misclassified. Review carefully.
```

Shown when `payload.containsAmbiguity === true` OR
`payload.headlineSeverity === 's3-critical'`.

### 2.2 Headline

```markdown
### TextUI Diff — <headline text>
```

`headline` comes directly from `DiffPRCommentPayload.headline`.

### 2.3 Group block

For each `DiffPRCommentGroup`:

```markdown
#### <axis label> — <highestSeverity badge>

<narrativeParagraph>

<finding row> × items
```

**Axis label mapping:**

| axis value | Label |
|---|---|
| `ambiguity` | Ambiguity |
| `structure` | Structure |
| `behavior` | Behavior |
| `permission` | Permission |
| `flow` | Flow |
| `state` | State |
| `event` | Event |
| `presentation` | Presentation |

**Severity badge:**

| severity | Badge |
|---|---|
| `s3-critical` | 🔴 Critical |
| `s2-review` | 🟡 Review |
| `s1-info` | 🔵 Info |
| `s0-minor` | ⚪ Minor |

### 2.4 Finding row

```markdown
- [<severity icon>] <label> `<eventId>` [↗ <evidenceLink>] [*(heuristic)*] [*(ambiguous)*]
```

- `evidenceLink`: rendered as `[view](<jumpTarget>)` when `evidenceJump` is non-null.
  Uses `jumpTarget` from `DiffReviewerEvidenceJump`.  Omitted silently when null.
- `*(heuristic)*`: shown when `heuristicDerived === true`.
- `*(ambiguous)*`: shown when `ambiguityMarker === true`.
- `eventId` is shown in backtick monospace for traceability.

**Severity icon:**

| severity | Icon |
|---|---|
| `s3-critical` | 🔴 |
| `s2-review` | 🟡 |
| `s1-info` | 🔵 |
| `s0-minor` | ⚪ |

### 2.5 Truncation notice

When `truncation.truncated === true`:

```markdown
<details>
<summary>… and <N> more items (click to view all)</summary>

[View full diff result](<detailLink>)
</details>
```

When `detailLink` is null, the `[View full diff result]` line is omitted.

### 2.6 Footer

```markdown
---
*Generated by `textui-diff-core` · schema `diff-result-external/v0`*
```

Uses `payload.sourceRef.engine` and `payload.sourceRef.schemaVersion`.

---

## 3. Rendering modes

| Mode | Description |
|---|---|
| `concise` | Headline + attention banner + group summary lines (no per-item rows) |
| `full` | Full structure as described above |

In `concise` mode, the group block renders only the axis label, severity badge,
and narrative paragraph.  Individual finding rows are omitted; the truncation
notice is replaced with "N total findings".

The mode is selected by the caller (G2 integration layer); the formatter
receives it as a parameter.

---

## 4. Formatter function signature

```ts
type DiffPRCommentRenderMode = 'concise' | 'full';

function renderDiffPRComment(
  payload: DiffPRCommentPayload,
  mode?: DiffPRCommentRenderMode  // default: 'full'
): string
```
